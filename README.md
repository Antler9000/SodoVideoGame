# TheseusAtLast
- **장르**  
실시간 전술 캠페인 게임

- **특징**  
조작 유닛 규모를 소대급으로 제한하여, 조작 부담을 줄이고 각 유닛의 기능과 개성 그리고 스토리에 집중함

- **기술**  
상용 엔진 대신, Win32와 D3D12로 제작

- **목표**  
올해 2026년 3분기까지 조명, 환경, 유닛 기능이 담긴 데모 미션 제작

- **현재**  
최소한의 윈도우 3D 게임 형태를 갖추기 위한 탐색 구현 단계 중

## 3. 제작 동기
요소들이 직교적으로 조합되며 다가올 때, 새로운 가능성과 즐거움을 느낀다고 생각합니다.

실시간 전술 게임은 그러한 요소들을 넓은 탑다운 뷰, 맵의 관경, 여러 유닛들, 전쟁이라는 동기적 배경을 통해 담아냅니다.

컴퍼니 오브 히어로즈는 그러한 게임이지만 많은 후반전 유닛 수로 인해 조작이 부담스럽고, 유닛이 표준화되며, 스토리가 게임성과 결합되지 못하는 점이 아쉬웠습니다.

현재까지 학습한 D3D12의 기본적인 렌더링, 지형 생성, 화면 공간 선택 등의 기법을 이용하면 실시간 전술 게임을 제작할 수 있다고 생각했고,

따라서 유닛 수를 줄여 이들의 기능과 개성, 그리고 스토리와의 결합에 집중한 실시간 전술 게임을 만들어보고자 본 프로젝트를 시작하게 되었습니다.

<br>




## 4. 기능 구상
- **조명 기능**
  | 종류 | 계산 방식 |
  |---|---|
  | 반영 조명(Specular Light) | 프레넬 상수 + 표면 거칠기 상수 + 쉐도우 맵 + 법선 맵 |
  | 분산 조명(Diffuse Light) | 분산 맵 + 쉐도우 맵 + 법선 맵 |
  | 주변 조명(Ambient Light) | SSAO |

  일단 학습 경험이 있는 다이렉트X 12 입문 도서에서의 조명 방식을 사용하여 기초적인 시각 경험을 제공할 예정입니다.

  추가적인 렌더링 기법은 추가 학습에 따라 점진적으로 도입할 예정입니다.

  <br>

  **(관련 상호작용)**
  - 카메라 조작
  - 포구 화염 효과
  - 그래픽 설정 GUI

<br>

- **소리 기능**
  | 종류 | 녹음 방식 |
  |---|---|
  | 환경 소리 | 미정 |
  | 차량 소리 | 미정 |
  | 인물 대사 | 미정 |
  | 배경 음악 | 미정 |

  본 리포지토리의 라이센스에 적합한 생성 모델을 결정 중에 있습니다.  

  ElevenCreative의 기능과 라이센스를 확인 후 결정지을 예정입니다.

  <br>

  **(관련 상호작용)**
  - 차량 기동 및 전투 효과음
  - 유닛 선택 대사
  - GUI 클릭 소리
  - 음량 설정 GUI

<br>

- **환경 기능**   
  | 종류 | 제작 방식 |
  |---|---|
  | 초목 | ChatGPT |
  | 바위 | 위와 동일 |
  | 건물 | 위와 동일 |
  
  일단 출력물의 라이센스 제약이 적은 ChatGPT로 제작한 기초적인 에셋을 관상용 목적으로 배치할 계획입니다.
  
  <br>

  **(관련 상호작용)**
  - 에셋이 유닛들의 기동, 시야, 사격을 제한함

<br>

- **유닛 기능**  
  | 종류 | 구성 방식 |
  |---|---|
  | 차량 | 2차 세계대전의 기갑 차량 |
  | 인물 | 각 차량의 차장 총 4인 |

  기갑 차량 중에선 중후한 외관을 뽐내는 영국과 독일의 2차 세계대전 기갑 장비들을 등장시키려 합니다.

  주인공을 포함한 4명의 등장 인물이 각 유닛을 나타내어, 유닛의 대사에 개성을 부여하려 합니다.

  <br>

  **(관련 상호작용)**
  - 마우스을 통한 유닛 선택 및 분대 지정
  - 이동 명령, 공격 명령, 능력 사용 명령
  - 전술 및 전략 자원 관리
  - 위 상호작용의  GUI

<br>

- **스토리 기능**  
  | 종류 | 전달 방식 |
  |---|---|
  | 컷씬 | 미션 시작과 끝 |
  | 대사 | 미션 중 |
  | 플롯 | 선형적 진행 |
  | 테마 | 없음 |

  미션마다 도입용 컷씬과 종료 컷씬을 제작할 계획입니다.

  미션 중에는 유닛의 대사를 통해 진행을 이끌어 나갈 예정입니다.

  스토리를 플레이어의 행동에 따라 분기하기보다는 하나의 선형적인 스토리를 더 잘 제시하는 것에 집중하려 합니다.

  전달하고자 하는 테마는 없습니다.
  
  <br>

  **(관련 상호작용)**
  - 적 동작 스크립트
  - 캠페인 시작, 세이브, 로드 GUI

## 5. 설계 사항
- **폴더 구조**

<br>

- **파일 포맷**  
  아직 설계되지 않음

<br>

- **빌드 관리**  
  일단 Visual Studio로만 빌드 설정을 관리하고 있습니다.

<br>

- **입력 방식**  
  일반적인 윈도우 메시지 처리 방식을 사용하고 있습니다.

<br>

- **사운드 방식**  
  아직 설계되지 않음

<br>

- **제시 방식**  
  제시(Present) 모드는 마이크로소프트의 최적화 권장을 따라 flip 모드를 사용할 계획입니다.

  SetMaximumFrameLatency(..)의 인자 값은 2로 두려고 합니다.  
  2로 설정한다면 하나의 present가 이미 큐잉된 상태에도 추가 프레임을 렌더링 할 수 있고,  
  프레임 지연은 크게 증가하지 않으리라 예상하기 때문입니다.  
  다만 실제 성능을 확인하고서 필요시 값을 교체할 예정입니다.  

<br>

- **GUI 구성**  
  심미성이 부족하더라도, imgui를 이용하여 단순한 GUI를 제공하려고 합니다.  
  자체적인 GUI를 설계 및 구현하는 것은 힘들다고 판단하였습니다.
